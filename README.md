# 100 Days of Python

A beginner-friendly Python course built as 100 daily lessons.

This course starts from the beginning: what programming is, how Python runs code, how to read mistakes, and how to write small programs with confidence. It then builds steadily toward functions, files, APIs, modules, classes, tests, Git, and a final capstone project.

The goal is not to rush through syntax. The goal is to help a beginner think like a programmer, one practical lesson at a time.

## Who This Course Is For

This course is for:

- complete beginners
- students learning Python for the first time
- self-taught learners who want structure
- anyone who has watched tutorials but still struggles to build programs alone
- learners who want practice, debugging habits, and real projects

No prior programming experience is required.

## How The Course Works

Each day is a standalone Markdown lesson in the `days/` folder.

A typical lesson includes:

- a clear goal
- the main idea
- a beginner mental model
- working examples
- common mistakes
- debugging practice
- exercises
- a small challenge
- a short bridge to the next day

The course is designed to be typed, run, broken, fixed, and understood.

## Requirements

You need:

- Python 3
- a code editor
- a terminal or command prompt
- patience with mistakes

A virtual environment is recommended:

```bash
python -m venv .venv
```

On Windows:

```powershell
.\.venv\Scripts\Activate.ps1
```

On macOS or Linux:

```bash
source .venv/bin/activate
```

## Suggested Pace

Use one lesson per day if you want the full 100-day structure.

For each day:

1. Read the lesson.
2. Type the code yourself.
3. Run the examples.
4. Break something on purpose.
5. Fix it.
6. Complete the exercises.
7. Write down one thing that became clearer.

Some days will take less than an hour. Some project days may take longer. That is normal.

## Learning Philosophy

This course teaches Python through small, steady steps.

It avoids dumping too many ideas at once. New concepts are introduced only after the learner has enough context to use them. Mistakes are treated as part of the work, not as failure.

By the end, a learner should be able to:

- read and write Python programs
- use variables, conditions, loops, collections, and functions
- work with files and structured data
- handle common errors
- organize code into modules
- use classes for simple real-world models
- write basic tests
- refactor code with more confidence
- plan and build a small complete project

## Final Outcome

By Day 100, the learner will have built a capstone project and practiced the habits that matter after the course ends:

- planning before coding
- debugging carefully
- reading documentation
- testing small pieces
- organizing files clearly
- improving a project in stages

This is not the end of learning Python. It is the point where the learner is ready to keep building.
